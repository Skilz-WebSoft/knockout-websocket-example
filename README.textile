h1. Knockout WebSockets Example

A static page with the javascript lib "Knockout":http://knockoutjs.com/ and "jQuery Templating plugin":http://api.jquery.com/category/plugins/templates/. It posts new items to a "Sinatra app":http://www.sinatrarb.com/, which in turn saves it to a SQLlite db and then pushes notifications via the message queue "Beanstalk":http://kr.github.com/beanstalkd/. An EventMachine listens to the Beanstalk queue ("EM-Jack":http://github.com/dj2/em-jack) and pushes the json data to its WebSocket subscribers ("EM-WebSocket":http://github.com/igrigorik/em-websocket). Knockout then addes the items to it's observable array, which is rendered with the jquery templating engine. 

h2. Usage

Install and start "Beanstalk":http://kr.github.com/beanstalkd/

bc. sudo apt-get install beanstalkd
beanstalk -d

Download the code

bc. git clone git://github.com/carlhoerberg/knockout-websocket-example.git
cd knockout-websocket-example

Install the dependencies

bc. bundle install

Start the Sinatra app

bc. rackup

Start the WebSocket server

bc. ruby socket.rb

Browse to http://localhost:9292/ in two different browser and start adding and modifing entries and see how they appear  instantly in the other browser.
